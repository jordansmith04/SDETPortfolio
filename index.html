What I Do

Software Test Engineering is about looking for edge cases, thinking like a malicious user, finding and fixing bugs quickly, and ensuring a perfect experience for our users.

Some of the approaches I take are:

Positive / "Happy" testing

Checking that the application behaves and functions appropriately when users act as expected

Negative testing

Testing that the application handles errors gracefully when end users don't use the application as intented

Blackbox testing

Attempting to break the application without knowledge of the code inside through UI and browser developer tools

White box testing

Breaking the application from the inside. Checking error handling and validation

Performance testing

Load testing the application, checking response times, and making sure the SLA is met

Regression testing

Ensuring the functionality we complete today isn't being broken tomorrow
